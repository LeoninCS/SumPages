$ fork-skill 使用指南 GitHub Repository

One-link webpage cloning

给 Codex 一个 网页链接， 复刻前端体验。

fork-skill 会镜像源页面、采集浏览器证据、生成交互矩阵，并推动 Codex 在当前项目中重建 UI、动画、响应式布局和可见交互。

用途

适合的任务

用自然语言表达复刻、仿站、还原、照着做、clone、copy、rebuild、recreate 等意图，并附上网页 URL。

工作方式

先镜像源页面，再用 Playwright 和 DevTools 采集截图、DOM、样式、资源和交互证据，最后用验证报告驱动修复。

安装

在 fork-skill 仓库里运行安装脚本，然后在目标前端项目中安装运行依赖。

安装 skill 
node scripts/install.mjs
安装目标项目依赖 
npm install -D playwright pixelmatch pngjs
npx playwright install chromium
brew install wget

一条指令使用

在目标项目里打开 Codex，用自然语言描述复刻意图并附上真实 URL。

中文 prompt 
复刻 https://example.com
我想复刻 https://example.com
复刻这个网站 https://example.com
仿站 https://example.com
还原 https://example.com
照着 https://example.com 做一个
English prompt 
Recreate https://example.com
Copy https://example.com
Rebuild https://example.com
Clone https://example.com into this project

默认规则

  • Source 来自提示词 URL
  • Target route: /
  • Desktop: 1440x900
  • Mobile: 390x844
  • Evidence: .fork-skill/evidence/latest
  • Report: .fork-skill/reports/latest
  • Threshold: 0.02

手动流程

初始化镜像和 runbook

生成源页面镜像、交互矩阵和可执行命令清单。

init 
node "$HOME/.codex/skills/fork-skill/scripts/one-link-init.mjs" --url https://example.com/

启动源页面镜像服务

runbook 会给出精确入口，默认服务端口是 4173。

source server 
python3 -m http.server 4173 --directory .fork-skill/source

采集源页面证据

记录源页面的截图、样式、资源、文本和动画帧。

source evidence 
node "$HOME/.codex/skills/fork-skill/scripts/capture-evidence.mjs" \
  --source http://127.0.0.1:4173/example.com/index.html \
  --out .fork-skill/evidence/source-pass \
  --viewports desktop=1440x900,mobile=390x844

采集 source/target 双端证据

实现复刻页面后，对源页面和目标页面进行同一套截图与交互采集。

paired evidence 
node "$HOME/.codex/skills/fork-skill/scripts/capture-evidence.mjs" \
  --source http://127.0.0.1:4173/example.com/index.html \
  --target http://127.0.0.1:5173/ \
  --out .fork-skill/evidence/latest \
  --viewports desktop=1440x900,mobile=390x844 \
  --interactions .fork-skill/interactions.json

验证并修复

打开报告里的 diff、文本漂移和修复优先级，按最大可见差异逐项处理。

validate 
node "$HOME/.codex/skills/fork-skill/scripts/validate-evidence.mjs" \
  --evidence .fork-skill/evidence/latest \
  --out .fork-skill/reports/latest \
  --threshold 0.02

交互矩阵

把关键 hover、click、fill、scroll 等状态写进 `.fork-skill/interactions.json`。

interactions.json 
[
  {"name":"nav-hover","action":"hover","selector":"nav a:first-child","wait":300},
  {"name":"menu-open","action":"click","selector":"button[aria-label='Menu']","wait":500},
  {"name":"form-fill","action":"fill","selector":"input[type='email']","value":"test@example.com","wait":300},
  {"name":"scroll-middle","action":"scroll","y":800,"wait":500}
]

证据产物

  • .fork-skill/source/manifest.json：镜像源页面入口候选。
  • .fork-skill/runbook.json：源 URL、本地源页面 URL、服务命令、采集命令和验证命令。
  • .fork-skill/interactions.json：交互矩阵。
  • .fork-skill/evidence/latest/source/**：源页面截图、动画帧、DOM、style dump、资源日志和交互结果。
  • .fork-skill/evidence/latest/target/**：目标页面对应证据。
  • .fork-skill/reports/latest/report.md：人工可读验证报告。
  • .fork-skill/reports/latest/diffs/**.png：截图差异图。

验收标准

视觉和响应式

桌面与移动端基础截图通过 `--threshold 0.02`，首屏、全页面顺序、间距、字体、色彩、阴影、边框和图片裁切保持一致。

交互和动画

关键交互 before/after 截图齐全，菜单、表单、滚动、焦点和动画帧有源页面与目标页面的配对证据。

内容和媒体

可见文本、导航、主按钮、表单、主要图片和视频状态与源页面匹配，远程数据用静态 fixtures 固化可见字段。

项目检查

目标项目的 format、lint、typecheck、test、build 按仓库现有命令执行，最终报告列出剩余差异和原因。
